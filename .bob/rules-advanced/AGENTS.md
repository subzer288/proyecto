# Advanced Mode Rules (Non-Obvious Only)

## Terraform Constraints

- **MUST run terraform commands from `infraestructura/` directory**, not project root
- S3 bucket names cannot contain underscores (AWS restriction, enforced in `variables.tf`)
- `storage_aws_iam_user_arn` and `storage_aws_external_id` have no defaults — must pass `-var-file=env/test.tfvars`
- Lambda zip is auto-generated by `archive_file` data source from `lambda/lambda_function.py` — never commit or manually create the zip
- Lambda timeout is 60s, SQS visibility timeout is 180s (3x) — intentional for retry handling

## Architecture Patterns

- Lambda uses `s3.copy_object()` for efficiency (not get + put)
- Lambda event is **double-nested**: `event["Records"]` → SQS record → `json.loads(body)["Records"]` → S3 record
- `DESTINATION_BUCKET` is injected as Lambda environment variable by Terraform, not hardcoded
- Both S3 buckets have KMS encryption; keys are in `kms.tf`
- Snowflake credentials loaded via `utils/credentials.py` from environment variables

## Snowflake Seeder Patterns

- `utils/credentials.py` returns `None` silently for missing env vars; `seeder.py` validates explicitly before connecting
- SQL built via f-string interpolation — string values need to be quoted inside the SQL string
- Custom name/date generators in `utils/` use static word lists, not external libraries

## Testing

- Tests run from `snowflake-seeder/` directory: `pytest`
- `test/conftest.py` inserts parent dir into `sys.path` — required for `utils/` imports to work from `test/`
- Single test: `pytest test/test_seeder.py::TestSeeder::test_main_creates_table_and_inserts_records`
- All Snowflake/AWS interactions mocked via `unittest.mock.patch`
